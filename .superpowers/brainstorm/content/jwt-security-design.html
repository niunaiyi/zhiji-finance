JWT 安全模式架构设计

短效 Access Token + 黑名单刷新机制

认证与刷新时序图
前端 (Client) --- 请求 --- 服务端 (API)
1. POST /login → 验证凭据
存储 Token (1h) ← 颁发 JWT (A)
... 1小时后 Token (A) 过期 ...
2. GET /data → Token 过期校验
捕获 401 错误 ← 返回 401 Unauthorized
3. POST /refresh → 废弃 A，生成 B
更新 Token (B) ← 返回 JWT (B)
4. 重试 GET /data → 验证 Token (B)

确认设计细节

1

启用黑名单 (推荐)

刷新后旧 Token 立即失效，安全性最高。

2

设置刷新宽限期

允许在刷新后的几十秒内旧 Token 依然有效，解决并发请求时的 401 问题。
